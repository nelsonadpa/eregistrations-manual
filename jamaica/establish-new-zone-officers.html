PHASE 1 — INTAKE

Step-by-Step: Documents Check

  1. 1 Navigate to Part B → click "File pending" application
  2. 2 Switch to the Documents tab
  3. 3 Click the first document link → enter document carousel
  4. 4 For each document: review → click "Yes" if valid, "No" if not
  5. 5 After all 33 documents validated → FORMDATAVALIDATIONSTATUS is set automatically
  6. 6 Switch to the Processing tab
  7. 7 Click Approve documents check
  8. 8 Confirm in the dialog → application moves to Phase 2
If the button stays disabled: Ensure every document has been reviewed ("Yes" clicked). All filestatus fields must be true.

PHASE 3 — EXTERNAL NOC

Agency Decision Process

All three agency roles follow the same steps:

  1. 1 Log in with your agency credentials → navigate to Part B
  2. 2 Open the application
  3. 3 Review the consultation package and relevant documentation
  4. 4 Select the "No objection" radio button on the form
  5. 5 Click Send decision to SEZA

The decision is transmitted back to SEZA for consolidation. If you have an objection, select the appropriate alternative option.

Corrections & Rejection

Handling send-backs and denials

Corrections Flow

When Can Corrections Be Requested?

PhaseTrigger
Phase 1 (Doc Check)Incomplete, illegible, or incorrect documents
Phase 5 (ARC)Application requires additional information

What Happens

  1. Officer selects "Send back for corrections"
  2. Application routes to Complementary Information (applicant-side)
  3. Applicant receives notification
  4. Applicant corrects and clicks "Validate send page"
  5. Application returns to the requesting step

Reviewing a Corrected Submission

  1. Application reappears with "File pending" status
  2. Open and review corrected documents/data
  3. If satisfactory → proceed with normal approval
  4. If still insufficient → send back again
After ARC send-back, selective routing only re-activates the affected evaluation domains, not the entire pipeline.

Rejection Flow

When to Reject

PhaseDecision Maker
Phase 5 (ARC)Application Review Committee
Phase 7 (Board)Board of Directors

Required Documentation

  • All evaluation notes and risk assessments recorded
  • Rejection reason documented in form fields
  • Status letter communicates decision to applicant

What the Applicant Sees

Upon rejection:

  • Notification via platform and/or email
  • Status letter with official decision
  • Reasons for rejection (if provided)
Rejections are final. Ensure all justifications are properly documented before proceeding.

Troubleshooting

Common issues and quick reference

Common Issues

Disabled Action Button

Cause: FORMDATAVALIDATIONSTATUS not set to true.

Fix: Ensure all required fields are filled. For Doc Check: validate every document ("Yes"). Save and refresh if needed.

EditGrid Blocking Submission

Cause: Rows in "new" or "editing" state.

Fix: Find open rows (different background), click the row save button (checkmark) for each one.

NOC — Upload Required

Cause: Not all consultation documents uploaded.

Fix: Check all "Browse" upload slots. Upload a PDF for each agency. Wait for completion, then save.

Agency — Radio Required

Cause: "No objection" radio not selected.

Fix: Find the decision radio button group. Select "No objection" (or alternative). Save the form.

More Issues

Page Redirects to List

Cause: Application not at your step, or incorrect role assignment.

Fix: Check "Active task" column. Verify your account role with administrator.

Nothing Happens After Click

Cause: Confirmation dialog may be behind the window, or system is processing.

Fix: Wait 10-15 seconds. Look for confirmation dialogs. Check for error notifications at top of page.
